API Information

The GET/HEAD endpoint of the api is at /recipes

You can also find all ingredients used on the site here /ingredients

(this can be useful for searching for recipes sometimes)

The POST/PUT/DELETE api endpoint is at /modify-recipes

Parameters:

  • name - finds recipes with this exact entire phrase
  • searchterm - finds recipes with titles that contain this phrase
  • ingredients - for searching for recipes with certain ingredients. Seperate them with commas!
  • limit - limits the number of results sent back. The default is no limit.
All of the parameters can be used at the same time for searching, but using the name parameter
at the same time as the other parameters would be redundant, since at this time
every recipe is required to have a unique name, so the name parameter will always come up
with just one recipe.

Using none of the parameters will send back every recipe.
When sending a post request, you need a name, an array of ingredients, and an array of steps.
Ingredients must each have a name, an amount, and a unit. A link is technically required but
you can leave it blank. Steps require a text attribute which holds the text of the step.
(This is to make it so vue can loop through the steps array).
